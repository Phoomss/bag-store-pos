# ระบบขายหน้าร้าน (POS) และจัดการคลังสินค้าสำหรับร้านขายกระเป๋า
## (Bag Store POS & Inventory Management System)

ระบบขายหน้าร้าน (Point of Sale) และระบบจัดการคลังสินค้าแบบครบวงจร ออกแบบมาเฉพาะสำหรับการดำเนินธุรกิจร้านขายกระเป๋า รองรับระบบสมาชิก การตัดสต็อกอัตโนมัติ การรายงานยอดขาย และการควบคุมสิทธิ์ผู้ใช้งาน (RBAC) พัฒนาด้วยสถาปัตยกรรม PHP MVC แบบ Custom โดยไม่พึ่งพิงเฟรมเวิร์กขนาดใหญ่ เพื่อให้ระบบทำงานได้อย่างรวดเร็วและมีประสิทธิภาพสูง

---

## 🌟 ฟีเจอร์เด่นของระบบ (Features)

*   **ระบบขายหน้าร้าน (POS System)**
    *   หน้าจอขายหน้าร้านแบบตอบสนองรวดเร็ว (Responsive UI) ค้นหาสินค้าด้วย SKU หรือบาร์โค้ด
    *   คำนวณเงินสด เงินทอน ส่วนลด และระบบคำนวณภาษีมูลค่าเพิ่ม (VAT 7%) อัตโนมัติ
    *   เชื่อมต่อระบบสมาชิก ค้นหาเบอร์โทรศัพท์ลูกค้า สะสมแต้ม และปรับระดับชั้นสมาชิกอัตโนมัติ
    *   รองรับช่องชำระเงินด้วย PromptPay QR โดยแสดงรูปภาพ QR Code ของร้านค้าที่อัปโหลดไว้ หรือสร้าง QR Code จากหมายเลขพร้อมเพย์โดยอัตโนมัติ
*   **ระบบจัดการสินค้าและแบรนด์ (Products & Brands Management)**
    *   จัดการข้อมูลประเภทกระเป๋า (Categories) เช่น กระเป๋าเป้, กระเป๋าเดินทาง, กระเป๋าเงิน
    *   จัดการข้อมูลยี่ห้อ/แบรนด์สินค้า (Brands) เช่น Herschel, Samsonite, Anello, Tumi, Fjallraven
    *   บันทึกข้อมูลสินค้า รายละเอียด รหัส SKU บาร์โค้ด สี วัสดุ ขนาด ราคาทุน ราคาขาย และราคาส่งเสริมการขาย (Promotion Price)
*   **ระบบจัดการคลังสินค้า (Inventory Management)**
    *   ติดตามสถานะจำนวนสินค้าในคลังแบบ Real-time พร้อมแจ้งเตือนเมื่อสินค้าใกล้หมด (Min Stock)
    *   ระบบบันทึกประวัติการเคลื่อนไหวของสินค้า (Stock Movements) สำหรับการตรวจสอบย้อนหลัง
    *   ระบบรับสินค้าเข้าคลัง (Purchases) และระบบปรับยอดสินค้าเสียหาย/สูญหาย (Stock Adjustments)
*   **ระบบการจัดการผู้ใช้งานและการควบคุมสิทธิ์ (Role-Based Access Control - RBAC)**
    *   แบ่งกลุ่มผู้ใช้งานออกเป็น 4 ระดับ: เจ้าของร้าน (Owner), ผู้ดูแลระบบ (Admin), แคชเชียร์ (Cashier) และพนักงานคลังสินค้า (Warehouse)
    *   ระบบบันทึกประวัติการทำรายการสำคัญของพนักงาน (Audit Logs) และประวัติการเข้าสู่ระบบ (Login History) เพื่อความปลอดภัย
*   **ระบบรายงานและค่าใช้จ่าย (Reports & Expenses)**
    *   รายงานสรุปยอดขายประจำวัน รายงานสินค้าขายดี และรายงานสรุปแยกรายพนักงานแคชเชียร์
    *   จัดการและลงบันทึกรายการค่าใช้จ่ายอื่น ๆ ของร้าน (เช่น ค่าเช่าที่ ค่าไฟ ค่าอุปกรณ์)
*   **การตั้งค่าระบบ (System Settings)**
    *   ตั้งค่าข้อมูลร้านค้า ชื่อร้าน ที่อยู่ เบอร์โทร ข้อความท้ายใบเสร็จรับเงิน อัตราภาษี การตั้งค่าส่งอีเมลแจ้งเตือน (SMTP) และ **การตั้งค่าอัปโหลดรูปภาพพร้อมเพย์ QR Code หรือหมายเลขพร้อมเพย์ประจำร้าน**

## 🛠️ เทคโนโลยีที่ใช้พัฒนา (Tech Stack)

*   **ภาษาและระบบหลัก (Core Back-end):** PHP 8.3 หรือสูงกว่า (สถาปัตยกรรม Custom MVC)
*   **ฐานข้อมูล (Database):** MySQL 8.0
*   **ส่วนการแสดงผล (Front-end):** HTML5, CSS3 (การจัดแต่งสไตล์แบบ Premium Theme), Vanilla JavaScript
*   **ระบบคอนเทนเนอร์ (Containerization):** Docker & Docker Compose
*   **การจัดการไลบรารี (Dependency Management):** Composer (ใช้งานร่วมกับ `vlucas/phpdotenv` สำหรับ Config สภาพแวดล้อม)

## 🚀 ขั้นตอนการติดตั้งและรันระบบ (Setup & Installation)

ระบบนี้รองรับการรันผ่าน **Docker** และ **Docker Compose** เพื่อความสะดวกในการจำลองสภาพแวดล้อมการทำงาน โดยมีขั้นตอนการตั้งค่าดังนี้:

### 1. ความต้องการของระบบ (Prerequisites)
เครื่องคอมพิวเตอร์ของคุณจำเป็นต้องติดตั้งโปรแกรมต่อไปนี้ก่อน:
*   [Git](https://git-scm.com/)
*   [Docker Desktop](https://www.docker.com/products/docker-desktop/)

### 2. ตั้งค่าไฟล์คอนฟิกสภาพแวดล้อม (.env)
คัดลอกไฟล์คอนฟิกตัวอย่าง `.env.example` ไปเป็น `.env`
```bash
cp .env.example .env
```
*(คุณสามารถเปิดไฟล์ `.env` เพื่อแก้ไขการตั้งค่าต่าง ๆ เช่น พอร์ตเชื่อมต่อ, ชื่อฐานข้อมูล หรือ อัตราภาษีได้ตามความต้องการ)*

### 3. รันระบบผ่าน Docker Compose
เริ่มทำงาน Docker Container ของเว็บเซิร์ฟเวอร์ (Apache+PHP) และฐานข้อมูล (MySQL):
```bash
docker-compose up -d --build
```
*   เมื่อรันเสร็จสิ้น เว็บไซต์จะเปิดให้บริการที่พอร์ต `8080` (ลิงก์: http://localhost:8080)
*   ฐานข้อมูล MySQL ภายนอกจะสามารถเข้าถึงได้ผ่านพอร์ต `3307` (รหัสผ่าน root: `root`)

### 4. ติดตั้ง Dependencies ของ PHP
สั่งให้ Composer ติดตั้งไลบรารีที่จำเป็นภายใน Container:
```bash
docker-compose exec app composer install
```

### 5. ติดตั้งโครงสร้างฐานข้อมูลและข้อมูลเริ่มต้น (Database Migration & Seeding)
รันสคริปต์สคีมาข้อมูลเพื่อสร้างตารางทั้งหมด และนำเข้าข้อมูลตัวอย่าง (รวมถึงบัญชีผู้ใช้ทดสอบ คอนฟิกร้านค้า และรายการสินค้า):
```bash
docker-compose exec app php database/migrations/init.php
```

## 🔑 บัญชีผู้ใช้งานระบบสำหรับทดสอบ (Default Credentials)

เพื่อความสะดวกในการทดสอบฟีเจอร์และระดับสิทธิ์ต่าง ๆ สามารถใช้บัญชีตัวอย่างด้านล่างนี้ในการเข้าสู่ระบบ:

*   **รหัสผ่านสำหรับทุกบัญชี:** `password`

| บทบาท (Role) | อีเมลสำหรับเข้าใช้งาน (Email) | สิทธิ์หลักในระบบ |
| :--- | :--- | :--- |
| **เจ้าของร้าน (Owner)** | `owner@bagpos.com` | เข้าถึงข้อมูลทั้งหมด ดูรายงานการเงิน กำหนดสิทธิ์พนักงาน |
| **ผู้ดูแลระบบ (Admin)** | `admin@bagpos.com` | จัดการข้อมูลสินค้า แบรนด์ สมาชิก และผู้ใช้งาน |
| **แคชเชียร์ (Cashier)** | `cashier@bagpos.com` | ใช้งานระบบขายหน้าร้าน (POS) คิดเงินลูกค้า สมัครสมาชิก |
| **พนักงานคลังสินค้า (Warehouse)** | `warehouse@bagpos.com` | จัดการสต็อกสินค้า รับสินค้าเข้าคลัง ปรับยอดสต็อกสินค้า |

## 📂 โครงสร้างโฟลเดอร์ของโครงการ (Folder Structure)

```text
bag-pos/
├── app/                      # โค้ดหลักของแอปพลิเคชัน (PSR-4 Autoloading)
│   ├── Controllers/          # คอนโทรลเลอร์ควบคุม Logic ของแต่ละโมดูล
│   ├── Core/                 # คลาสหลักพื้นฐาน (Router, DB Connection, Request/Response)
│   ├── Helpers/              # ฟังก์ชันช่วยเหลือต่าง ๆ (การจัดฟอร์แมตเงิน, วันที่)
│   ├── Middleware/           # ตรวจสอบการเข้าสู่ระบบและสิทธิ์การเข้าถึงหน้าเว็บ
│   ├── Repositories/         # คลาสจัดการติดต่อสืบค้นข้อมูลดิบกับ MySQL (Query)
│   └── Services/             # คลาสจัดการ Logic ทางธุรกิจของระบบ (Business Logic)
├── database/                 # สคริปต์จัดการฐานข้อมูล
│   ├── migrations/           # สคริปต์การทำ Migration ตารางฐานข้อมูล
│   └── schema.sql            # ไฟล์สคีมาโครงสร้างตารางและข้อมูลตั้งต้น (Seed)
├── public/                   # โฟลเดอร์สาธารณะ (DocumentRoot)
│   ├── css/                  # ไฟล์สไตล์สไตล์ชีทของหน้าเว็บ
│   ├── js/                   # สคริปต์ JavaScript หน้าบ้าน
│   ├── uploads/              # โฟลเดอร์เก็บรูปภาพอัปโหลด (สินค้า และการตั้งค่า/PromptPay QR)
│   └── index.php             # ไฟล์เริ่มต้นการทำงานของระบบ (Entry Point)
├── views/                    # ไฟล์โครงร่างเทมเพลตหน้าเว็บ (HTML/PHP Layouts)
├── Dockerfile                # คำสั่งติดตั้งสภาพแวดล้อมสำหรับ Docker Container
├── docker-compose.yml        # คอนฟิกรัน Services (Web App & MySQL Database)
└── composer.json             # จัดการแพ็กเกจไลบรารีของ PHP
```
